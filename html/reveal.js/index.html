Nagios

is a system and service monitoring tool.

Nagios

  • alerts you of problems
  • warns you of upcoming problems
  • helps you pinpoint the actual problem

Nagios Example

            
define host{
  use        mathforum-server,vm-server
  host_name  rhenium
  alias      rhenium.mathforum.org
  address    144.118.94.126
}

Nagios Example

            
define service{
  use                     mathforum-service
  service_description     mathforum-drmath
  host_name               mathforum
  check_command           check_http!-H $HOSTALIAS$ -u /dr.math/
}

Nagios Example

            
define service {
  use                     base-database
  service_description     database-disk-space
  notifications_enabled   1
  hostgroup_name          postgres-servers
  check_command           check_postgres_by_ssh!disk_space
}

Nagios tips

  • minimize alerts (dependencies, schedules, acknowledgements)
  • whenever something breaks, add a nagios check for it
  • handy checks
    • DNS: valid
    • DNS: expiry
    • SSL: expiry
    • time (ntp)
    • ssh, disk, load
    • database (use plugins)
    • web - check all important URLs

Puppet

is a provisioning/configuration management tool.

Puppet

  • declarative
  • idempotent
  • multi-platform: EL, debian, etc, (even windows...)
  • client/server OR standalone
  • built on top of facter

Puppet Example


file {"/etc/motd":
  content => "hi there",
}

group {"drupal":
  ensure => "present",
}

Puppet Example


package { "apache2":
  ensure => present,
}

service { "apache2":
  ensure  => running,
  require => Package["apache2"],
}

file { "php.conf":
  path    => "${apache::params::vdir}php.conf",
  content => template("php.conf.erb"),
}

Puppet Example


<%# php.conf.erb %>
DirectoryIndex index.php
<VirtualHost <%= vhost_name %>:<%= port %> >
  ...
</VirtualHost>

Puppet Example


exec { "setup-drupal":
  command => "drush dl drupal-7.14",
  creates => "/usr/local/apache2/htdocs/drupal-7.14",
  cwd     => "/usr/local/apache2/htdocs",
  require => Package["drush"],
}

Exec {
  path => [ "/bin/", "/sbin/", "/usr/bin/", "/usr/sbin/", "/usr/local/bin/" ]
}

Vagrant

is a tool that creates and provisions development virtual machines

Vagrant Uses

  • virtualbox
  • Puppet or Chef

Getting started


vagrant box add base http://files.vagrantup.com/lucid32.box
cd project
vagrant init
vagrant up
vagrant ssh

What happens

  • virtualbox vm is created or started
  • ports are forwarded (2222 -> 22, 8080 -> 80)
  • provisioner runs to set up the box
  • files from the project are mounted at /vagrant

Conclusion

We avoid

  • regressions by testing with Selenium and running tests frequently with Hudson
  • "works for me" by building consistently with Hudson and provisioning with Puppet and Vagrant
  • being reactive by catching bugs early (regressions) and monitoring with nagios
  • wasting time, yes
